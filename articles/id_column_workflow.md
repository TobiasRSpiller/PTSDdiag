# Joining diagnoses back to demographics with id_col

## The problem

The `PTSDdiag` workflow normalises your dataframe to a canonical
`symptom_1` through `symptom_20` form before optimisation. Demographics
(`patient_id`, `age`, `sex`, …) are dropped at that step, so the
optimised diagnosis cannot be joined back to the original dataframe to
answer questions like *“do participants meeting the optimised criterion
differ in age from those who don’t?”*

This vignette shows how to thread a participant identifier through the
workflow using the `id_col` argument so that per-row diagnoses can be
merged back to demographics.

## Set up a dataframe with demographics

``` r

library(PTSDdiag)
library(dplyr)

set.seed(123)
n <- 200
study_df <- data.frame(
  patient_id = sprintf("P%03d", seq_len(n)),
  age        = sample(20:70, n, replace = TRUE),
  sex        = sample(c("F", "M"), n, replace = TRUE),
  simulated_ptsd[seq_len(n), ],
  stringsAsFactors = FALSE
)
head(study_df[, 1:6])
#>   patient_id age sex S1 S2 S3
#> 1       P001  50   F  2  1  4
#> 2       P002  34   M  3  3  4
#> 3       P003  70   F  0  1  1
#> 4       P004  33   F  3  3  3
#> 5       P005  22   F  2  3  3
#> 6       P006  61   M  1  1  2
```

## Carry the ID through `rename_ptsd_columns()`

Pass the column name(s) of any participant-level fields you want to
preserve. The simplest pattern is to use a single participant identifier
and join everything else back at the end:

``` r

sym <- rename_ptsd_columns(study_df[, c("patient_id", names(simulated_ptsd))],
                           id_col = "patient_id")
names(sym)[1:5]
#> [1] "patient_id" "symptom_1"  "symptom_2"  "symptom_3"  "symptom_4"
```

`sym` now contains `patient_id` plus `symptom_1` through `symptom_20`.
`age` and `sex` stay in `study_df` and will be joined back via
`patient_id` at the end. If you wanted them to ride through the workflow
as well, pass them too: `id_col = c("patient_id", "age", "sex")`.

## Optimise — the ID rides along

``` r

res <- optimize_combinations(
  sym, n_symptoms = 6, n_required = 4, n_top = 2,
  show_progress = FALSE
)
head(res$diagnosis_comparison)
#>   patient_id PTSD_orig symptom_2_6_7_11_12_15 symptom_2_6_7_11_12_19
#> 1       P001      TRUE                   TRUE                   TRUE
#> 2       P002      TRUE                   TRUE                   TRUE
#> 3       P003      TRUE                   TRUE                   TRUE
#> 4       P004      TRUE                   TRUE                   TRUE
#> 5       P005      TRUE                   TRUE                   TRUE
#> 6       P006      TRUE                  FALSE                   TRUE
```

`res$diagnosis_comparison` carries `patient_id` as its first column.
Every diagnostic column (`PTSD_orig`, the optimised combinations) is
aligned to the same `patient_id`.

## Join diagnoses back to demographics

``` r

final <- study_df %>%
  select(patient_id, age, sex) %>%
  left_join(res$diagnosis_comparison, by = "patient_id")
head(final)
#>   patient_id age sex PTSD_orig symptom_2_6_7_11_12_15 symptom_2_6_7_11_12_19
#> 1       P001  50   F      TRUE                   TRUE                   TRUE
#> 2       P002  34   M      TRUE                   TRUE                   TRUE
#> 3       P003  70   F      TRUE                   TRUE                   TRUE
#> 4       P004  33   F      TRUE                   TRUE                   TRUE
#> 5       P005  22   F      TRUE                   TRUE                   TRUE
#> 6       P006  61   M      TRUE                  FALSE                   TRUE
```

Now you can compare demographics by diagnostic status. The column
generated by the best optimised combination is named after its symptom
indices (e.g. `symptom_3_6_10_15_17_19`); pick the first non-`PTSD_orig`
diagnostic column for the demonstration:

``` r

combo_col <- setdiff(grep("^symptom_", names(final), value = TRUE),
                     paste0("symptom_", 1:20))[1]

final %>%
  group_by(.data[[combo_col]]) %>%
  summarise(n = n(), mean_age = mean(age), pct_female = mean(sex == "F"))
#> # A tibble: 2 × 4
#>   symptom_2_6_7_11_12_15     n mean_age pct_female
#>   <lgl>                  <int>    <dbl>      <dbl>
#> 1 FALSE                     13     48.4      0.692
#> 2 TRUE                     187     45.8      0.588
```

## The same trick works for validation

The ID rides through
[`holdout_validation()`](https://tobiasrspiller.github.io/PTSDdiag/reference/holdout_validation.md)
and
[`cross_validation()`](https://tobiasrspiller.github.io/PTSDdiag/reference/cross_validation.md)
too — useful because those functions reshuffle rows and positional
`cbind` would silently misalign your data.

``` r

ho <- holdout_validation(sym, train_ratio = 0.7, seed = 42,
                         n_symptoms = 6, n_required = 4, n_top = 2)
head(ho$without_clusters$test_results)
#>    patient_id PTSD_orig symptom_1_2_6_9_12_13 symptom_1_2_9_12_13_20
#> 7        P007      TRUE                  TRUE                   TRUE
#> 8        P008      TRUE                  TRUE                   TRUE
#> 10       P010      TRUE                  TRUE                   TRUE
#> 11       P011      TRUE                 FALSE                  FALSE
#> 17       P017      TRUE                  TRUE                   TRUE
#> 19       P019      TRUE                  TRUE                   TRUE
```

Each row of `test_results` carries the `patient_id` it came from, so a
`left_join` from demographics recovers the test fold exactly.

``` r

study_df %>%
  select(patient_id, age, sex) %>%
  inner_join(ho$without_clusters$test_results, by = "patient_id") %>%
  head()
#>   patient_id age sex PTSD_orig symptom_1_2_6_9_12_13 symptom_1_2_9_12_13_20
#> 1       P007  69   M      TRUE                  TRUE                   TRUE
#> 2       P008  62   M      TRUE                  TRUE                   TRUE
#> 3       P010  33   M      TRUE                  TRUE                   TRUE
#> 4       P011  44   M      TRUE                 FALSE                  FALSE
#> 5       P017  47   M      TRUE                  TRUE                   TRUE
#> 6       P019  48   M      TRUE                  TRUE                   TRUE
```

## Reserved column names

A small set of column names is reserved by the workflow and cannot be
used as `id_col`: `symptom_1` through `symptom_20`, `total`,
`PTSD_orig`, `PTSD_icd11`, `PTSD_caps5`, `PTSD_pcl5`, `.strata`, and any
name matching the combination-output pattern
(e.g. `symptom_1_6_8_10_15_19`). Rename your column before calling
[`rename_ptsd_columns()`](https://tobiasrspiller.github.io/PTSDdiag/reference/rename_ptsd_columns.md)
if it collides.
