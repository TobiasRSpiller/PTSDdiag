## Submission

This is an update of the CRAN package PTSDdiag (current CRAN version: 0.1.0;
this submission: 0.4.1). Highlights since 0.1.0:

* Multi-scenario optimization (`compare_optimizations()`) with summary
  tables, a symptom-frequency heatmap, and ICD-11 / CAPS-5 benchmarks.
* A complete multi-site validation workflow: JSON export/import of symptom
  definitions, `as_definitions()` to convert received files, evaluation
  against an external reference standard (e.g. a clinician CAPS diagnosis)
  and tidy metric tables via new `evaluate_definitions()` arguments, an
  exhaustive per-combination scoring companion (`score_all_combinations()`),
  and a one-pass input check (`check_pcl5_data()`).
* New `score_by = "balanced_accuracy"` optimization criterion, now the
  default; balanced accuracy is reported in all performance tables.
* Error messaging migrated to cli, an expanded test suite (including
  regression pins that keep 0.3.x outputs identical), a CITATION file, and
  five vignettes that build quickly on small subsets of the bundled data.

See NEWS.md for the complete changelog.
